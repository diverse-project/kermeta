Description :
-------------
This project contains the model of the Kermeta language as ecore models.

When the first kermeta iteration will be completed, this project should be refactored.

How the packages are organized :
--------------------------------

--------------------------------------------------
         Package global principles :              
--------------------------------------------------

- /dev : put here any code that is used to construct the language
- [TODO]


--------------------------------------------------
         Package description :              
--------------------------------------------------

- ast : 

    * Contains the java representation of the AST (generated by EMF)

- loader : 

    * Classes that load a kermeta unit from a text representation, or (another
    one) of kermeta
    
    - emfatic :
        
        * Class that transform a ECore representation to a Kermeta
        representation in 4 passes.
    
    - expression (FIXME) : 

        * Contains the parser for a single expression  (-> move to parser?)

    - km
    
       * Contains the classes that load a Kermeta Unit from a Kermeta-export?
       representation (XMI?).

    - kmt

        * Contains the classes that load a Kermeta Unit from a KMT/Text
        representation

- exporter : 

    * Contains classes to export a Kermeta object
        to another one (a text format (.kmt) for now -- later : ecore, xmi?)

- parser :

    * The AST kermeta.ast (generated by emfatic from KermetaLext.g file)
    * The parser classes generated by emfatic
    
- utils :

    * Contains methods that add facilities
