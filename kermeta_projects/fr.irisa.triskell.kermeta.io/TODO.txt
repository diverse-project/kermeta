Definition of the lambda expression :
There is a redundant syntax : 

function { parametres | appel(avec les parametres)} 
or : 
{ parametres | appel(avec les parametres) }

In the first case, function is used to "define" a lambda expression and affect an element of type 
FunctionType. Example : 
	result : FunctionType
	result := function { a | not(a) }
In the second case, it is used to "call" a lambda expression, which stands as an argument :
	acall( { a | not(a) } )

In a non functional language, we need a syntax to specify a lambda expression : here,
we have" function" and the curlies...So it is too much!
