KET Comment Syntax Reference

KET templates may contain comments between the characters <%-- and --%>. Comments have no impact on the execution of the template, except that they may influence whitespace stripping rules. KET comments are copied to the generated kermeta class as kermeta line comments.

KET templates accept two special tags in the first non-blank line of a comment. The tag '@header' will cause the comment to be emitted as the file header comment for the generated kermeta class. This can be useful for inserting copyright notices into the generated kermeta code. The tag '@class' will cause the comment to be emitted as the class kermeta doc comment for the generated kermeta class.

Valid comments

Comments may span several lines, and may contain any text. The following are examples of a KET comments:

  
    <%-- @header
    This comment will appear as the file header comment 
    in the generated kermeta code
     --%>
    <%-- 
    @class
    This comment will appear as the kermeta class doc comment 
    in the generated kermeta code
     --%>
     <%-- This comment will not appear in the template output --%>
     <%-- This directive is not used 
     <%@taglib id="org.eclipse.KET.controlTags" prefix="cc"%>
     --%>

Invalid comments

Comments may not appear within other KET elements. The following are illegal comments:

<%@taglib id="org.eclipse.KET.controlTags" <%-- illegal comment --%>
          prefix="cc" %>

Escaping comment characters

To emit the characters <%-- in a templates output, enter <\%--. To emit --%>, enter --%\>.

The following is an example of an escaped KET comments:

<\%-- this will show in the template output --%\>
