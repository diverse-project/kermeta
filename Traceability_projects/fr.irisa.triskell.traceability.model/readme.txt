Procedure to generate a new version of the model if you change the ecore model.
	* remove the traceability.ecore.genmodel
	* generate the model using omondo
	* modify traceability.ecore.genmodel
		- Model/Model Directory changed from /fr.irisa.triskell.traceability.model/src to /fr.irisa.triskell.traceability.model/build/java
		- remove the ecore package (no name in the tree of the genmodel)
		- Model Plugin-ID must be changed to fr.irisa.triskell.traceability 
		- the package Tracability/All/Base Package must be fr.irisa.triskell
		- edit plugin class changed from traceability.provider.TraceabilityEditPlugin
			to fr.irisa.triskell.traceability.provider.TraceabilityEditPlugin
		- editor plugin class changed from traceability.presentation.TraceabilityEditorPlugin
			to fr.irisa.triskell.traceability.presentation.TraceabilityEditorPlugin
	* remove the src/traceability folder (generated by the generate model by omondo)
	* remove /src/ecore/impl /src/ecore/util /src/ecore/Factory.java /src/ecore/Package.java
	* in Omondo, or on the new genmodel, generate all
	* in each of the edit and editor projects, change the plugin.xml
		- change the version according to your new version
		- change the provider name to www.inria.fr
	* eventually recompile the java files

The file build.xml was generated by the PDE and can be safely removed and regenerated.

the dist folder contains a ready to deploy version of the EMF plugins (model, edit and editor)
