Introduction to UML2 MDK

Kermeta UML2 Model development Kit contains various kind of code: some helper classes and some transformations that apply to UML2 metamodel.

If you don't use UML Profiles, to use these functionalities you simply have to add the corresponding require statement to your kermeta code. Then, the outline and the KermetaDoc views should help you to use its content.

If you use UML Profiles, due to the special implementation that have been done in Eclipse EMF, there are some special tricks that you should look at in the corresponding section.

Short description of the main available require for Profiles

require "platform:/plugin/org.kermeta.uml2.profiles.model/src/kmt/profiledResourceHelper/ProfiledResourceHelper.kmt"
This helper class provides some search functions useful to manipulate profiles (especially retrieving stereotypes etc, ...)
require "platform:/plugin/org.kermeta.uml2.profiles.model/model/ecore/Ecore.profile.ecore"
ecore version of Ecore.profile.uml. This is (indirectly) needed by most of the profile manipulation.
require "platform:/plugin/org.kermeta.uml2.profiles.model/model/ecore/Standard.profile.ecore"
This file contains a transformation class that allows to transform your Ecore model and apply Design Patterns to it. Provided Design Pattern (and variants) are : Visitor, Command, Builder and Observer.
require "platform:/plugin/org.kermeta.uml2.profiles.model/src/kmt/persistence/UMLResource.kmt"
Aspect for Kermeta persistence in order to support complex profiles (for example using nested profiles, ...). Simple profiles doen't need it.
require "platform:/plugin/org.kermeta.uml2.profiles.model/src/kmt/UMLProfile2Ecore/UMLProfile2Ecore.kmt"
This file contains an experimental transformation class that allows to create an Ecore model from an UML model.
